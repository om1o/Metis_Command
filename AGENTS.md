# Metis Agent Instructions

Metis is now a desktop-first automation operator.

Current product direction:
- Desktop first.
- Browser-control first MVP.
- Automation-first product surface.
- Manager-led execution model.
- Safe mode default.
- Per-job auto mode only.

What to prioritize:
- Browser cockpit.
- Automations board.
- Inbox with real event history.
- Manager policies for browser control, approvals, and limits.
- Local auth, local files, local browser runtime, and desktop shell reliability.

What is not the current priority unless required by the browser MVP:
- Chat-first product work.
- Code workspace features.
- Plugin-store-first work.
- Image, video, and voice expansion.
- Broad OS-control work outside the browser.

Execution rules:
- Read the code before editing it.
- Keep changes tightly scoped to the active browser automation MVP.
- Do not reintroduce the old chat/code/media roadmap.
- Commit coherent work and push it to GitHub.
- Tell the truth about blockers and failed checks.
